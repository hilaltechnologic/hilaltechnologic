## 🚀 Tip #1: Optimasi Gambar - The Game Changer

Gambar biasanya jadi biang keladi website lambat. Ini cara ngatasinnya:

### Format Gambar Modern

```html
<!-- Gunakan format modern -->
<picture>
  <source srcset="hero.avif" type="image/avif">
  <source srcset="hero.webp" type="image/webp">
  <img src="hero.jpg" alt="Hero image" loading="lazy">
</picture>
```

### Lazy Loading

```html
<!-- Native lazy loading -->
<img src="image.jpg" alt="Description" loading="lazy">
```

```javascript
// Atau pakai Intersection Observer
const images = document.querySelectorAll('img[data-src]');
const imageObserver = new IntersectionObserver((entries, observer) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      img.classList.remove('lazy');
      observer.unobserve(img);
    }
  });
});

images.forEach(img => imageObserver.observe(img));
```

**Tools untuk Optimasi:**
- TinyPNG/TinyJPG
- Squoosh.app
- ImageOptim (Mac)

## 📦 Tip #2: Minify dan Compress Assets

### CSS dan JavaScript Minification

```css
/* Before - CSS biasa */
.header {
    background-color: #ffffff;
    padding: 20px 15px;
    margin-bottom: 30px;
}

/* After - CSS minified */
.header{background-color:#fff;padding:20px 15px;margin-bottom:30px}
```

### Gzip Compression

```javascript
// Express.js dengan compression
const compression = require('compression');
const express = require('express');
const app = express();

```

```nginx
# Nginx config
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript application/javascript;
```

**Tools:**

## 🌐 Tip #3: Leverage Browser Caching

Cache itu kayak memory jangka pendek browser. Sekali download, nggak perlu download lagi.

```javascript
// Express.js cache headers
app.use(express.static('public', {
  maxAge: '1y', // Cache static files for 1 year
  etag: false
}));
```

```nginx
# Nginx cache config
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

### Service Worker untuk Advanced Caching

```javascript
// service-worker.js
const CACHE_NAME = 'my-site-cache-v1';
const urlsToCache = [
  '/',
  '/styles/main.css',
  '/scripts/main.js'
];

self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(urlsToCache))
  );
});

self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(response => response || fetch(event.request))
  );
});
```

## 🔄 Tip #5: Reduce HTTP Requests

Setiap request = round trip ke server. Makin sedikit makin cepat.

### CSS Sprites

```css
/* Gabungin multiple icons jadi satu file */
.icon {
    background-image: url('sprites.png');
    background-repeat: no-repeat;
}

.icon-home { background-position: 0 0; }
.icon-user { background-position: -20px 0; }
.icon-settings { background-position: -40px 0; }
```

### Inline Critical CSS

```html
<!-- Critical CSS langsung di HTML -->
<style>
  .hero { background: #000; color: #fff; height: 100vh; }
  .nav { position: fixed; top: 0; width: 100%; }
</style>

<!-- Non-critical CSS di-load async -->
<link rel="preload" href="styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
```

## 🗄️ Tip #6: Database Query Optimization

Backend yang lambat = frontend yang lambat juga.

### Index Database Fields

```sql
-- SQL: Tambah index di kolom yang sering di-query
CREATE INDEX idx_user_email ON users(email);
CREATE INDEX idx_post_created_at ON posts(created_at);
```

```javascript
// MongoDB: Index compound
db.users.createIndex({ "email": 1, "status": 1 });
```

### Query Optimization

```javascript
// Bad - N+1 Query Problem
const posts = await Post.findAll();
for (let post of posts) {
  post.author = await User.findById(post.authorId);
}

// Good - Eager Loading
const posts = await Post.findAll({
  include: [{ model: User, as: 'author' }]
});
```

## 🎨 Tip #7: Optimize CSS dan JavaScript Loading

### Critical Resource Hints

```html
<!-- Preload critical resources -->
<link rel="preload" href="critical.css" as="style">
<link rel="preload" href="hero-image.webp" as="image">

<!-- Prefetch for next page -->
<link rel="prefetch" href="/about">

<!-- DNS prefetch for external domains -->
<link rel="dns-prefetch" href="//fonts.googleapis.com">
```

### Async dan Defer Scripts

```html
<!-- Defer: Execute after HTML parsing -->
<script src="main.js" defer></script>

<!-- Async: Execute immediately when loaded -->
<script src="analytics.js" async></script>

<!-- Module scripts (modern browsers) -->
<script type="module" src="app.js"></script>
```

## 📱 Tip #8: Implement Progressive Loading

### Skeleton Screens

```css
.skeleton {
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: loading 1.5s infinite;
}

@keyframes loading {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

### Code Splitting

```javascript
// React lazy loading
import { lazy, Suspense } from 'react';

const LazyComponent = lazy(() => import('./LazyComponent'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <LazyComponent />
    </Suspense>
  );
}

// Webpack dynamic imports
const loadModule = async () => {
  const module = await import('./heavyModule.js');
  module.init();
};
```

## 🔧 Tip #9: Server-Side Optimizations

### HTTP/2 dan HTTP/3

```nginx
# Nginx HTTP/2 config
server {
    listen 443 ssl http2;
    server_name yoursite.com;
    
    # SSL config
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
}
```

### Response Compression

```javascript
// Node.js dengan Brotli compression
const express = require('express');
const compression = require('compression');

const app = express();

app.use(compression({
  level: 6,
  threshold: 1024,
  filter: (req, res) => {
    if (req.headers['x-no-compression']) return false;
    return compression.filter(req, res);
  }
}));
```

## 📊 Tip #10: Monitor dan Measure Performance

### Core Web Vitals

Yang harus lo pantau:
- **LCP (Largest Contentful Paint)** - < 2.5s
- **FID (First Input Delay)** - < 100ms  
- **CLS (Cumulative Layout Shift)** - < 0.1

### Performance Monitoring Tools

```javascript
// Web Vitals library
import {getCLS, getFID, getFCP, getLCP, getTTFB} from 'web-vitals';

getCLS(console.log);
getFID(console.log);
getFCP(console.log);
getLCP(console.log);
getTTFB(console.log);

// Performance Observer API
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log(entry.name, entry.duration);
  }
});

observer.observe({entryTypes: ['measure', 'navigation']});
```

**Tools untuk Testing:**
- **Google PageSpeed Insights**
- **GTmetrix**
- **WebPageTest**
- **Lighthouse** (built-in Chrome DevTools)
